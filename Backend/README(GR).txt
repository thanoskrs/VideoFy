STRUCTURE
 
MainServer:	
	1. Κάθε broker της εφαρμογής συνδέεται στον MainServer και ο MainServer αποθηκεύει σε κατάλληλη
	   δομή, την IP και την Port του κάθε Broker. Σκοπός είναι να στέλνονται μετά στους Appnodes.
	2. Κρατάει την πληροφορία για τα usernames και passwords των Αppnodes και αποθηκεύονται 
	   τοπικά στο credentials.txt για τις επόμενες συνδέσεις.
	3. Αποθηκεύει σε hashmap τον υπέθυνο broker για κάθε hashtag / channelName και θα στέλνει
	   την πληροφορία αυτή στους Consumers, όταν εκείνοι θέλουν να δουν κάποιο βίντεο.
Broker: 
	1. Κατά την εκτέλεση του κάθε broker, ο καθένας θα αρχικοποιεί το serverSocket του με μία τυχαία
	   διαθέσιμη Port.
	2. Θα στέλνει την ip του και την Port του στον MainServer.
	3. Κάθε φορά που θα ανεβαίνει ένα νέο βίντεο, ο κατάλληλος Broker θα συνδέεται εκ νέου στον 
	   MainServer, έτσι ώστε να του στέλνει τo hashtag ή το channelName που δέχτηκε καθώς και τα στοιχεία
	   του (ip/port).
	4. Είναι υπέυθυνος για την αποστολή των βίντεο από τους Pubishers στους Consumers.
Appnode:
	1. Όταν τρέξει ένας Appnode, θα μπορεί είτε να συνδεθεί -εάν έχει κάνει ήδη εγγραφή στο σύστημα- είτε
	   να κάνει εγγραφή στο σύστημα. Οπότε, γίνεται επικοινωνία με τον MainServer που έχει όλη αυτή την 
	   πληροφορία. Έχουμε ήδη αποθηκεύσει στο credentials.txt 3 χρήστες με τους κωδικούς τους, οπότε μπορεί
	   να γίνει log in με τα στοιχεία αυτά.
	2. Επίσης, από τον MainServer δέχονται και το hashmap με τους brokers και τα ip/port για να τα αξιοποιήσουν
	   στη συνέχεια.
	3. Ταυτόχρονα, για κάθε Appnode αρχικοποιείται το δικό του serverSocket <PublisherServer>. Σε αυτόν θα 
	   συνδέονται οι brokers για να πάρουν τα βίντεο του εκάστοτε Appnode και να τα στείλουν στον Consumer που 
	   τα ζήτησε.
	3. Ως Pubishers, όταν θέλουν να ανεβάσουν ένα νέο βίντεο, δημιουργούν επικοινωνία με τους κατάλληλους
	   brokers και τους ενημερώνουν για τα hashtags, καθώς και την ip/port του PublisherServer τους.
	4. Ως Consumers, αρχικά συνδέονται με τον MainServer για να πάρουν την πληροφορία για τα διαθέσιμα 
	   hashtags / channelName και τους υπέυθυνους brokers για καθένα από αυτά. Στη συνέχεια, μόλις επιλέξουν 
	   το κατάλληλο topic, δημιουργούν επικοινωνία με τον κατάλληλο broker για να πάρουν τα βίντεο.
PublisherServer:
	1. Αρχικοποιείται στην Appnode και σκοπός είναι να δέχεται αίτημα από κάποιον broker και εκτελεί την μέθοδο push, 
	   στέλνοντας τα βίντεο που έχουν ζητηθεί. 
	
	
	
	
ΕΚΤΕΛΕΣΗ **:
	1. Πρώτα, τρέχουμε τoν MainServer.
	2. Έχουμε αρχικοποιήσει 3 configurations των Broker. Τα Broker1, Broker2, Broker3. Τρέχουμε το κάθε 
	   configuration.
	3. Έχουμε αρχικοποιήσει 3 configurations των Appnode. Τα Αppnode1, Αppnode2, Αppnode3. Τρέχουμε τον κάθε
 	   Appnode. Να σημειώσουμε, ότι μπορούμε δυναμικά να δημιουργήσουμε όσους χρήστες θέλουμε. Οπότε, δεν 
	   υπάρχει περιορισμός για το πόσα configurations των Appnode θα τρέξουν.


** 
(1) Για να γίνει σωστά η επικοινωνία, αρκεί μόνο, οι Brokers και οι Appnodes να γνωρίζουν την IP και την Port
   του MainServer. Η πληροφορία είναι αποθηκευμένη στο αρχείο MainServer.txt, όπου σαν IP έχουμε την local "127.0.0.1" 
   και ένα προκαθορισμένο Port. Εάν όμως, κάποια από τα components τρέξουν σε διαφορετικούς υπολογιστές, θα πρέπει να 
   ενημερωθεί το MainServer.txt και να αλλάξει η IP, βάζοντας την IP του υπολογιστή που θα τρέξει ο MainServer.
	
(2) Εάν για κάποιο λόγο δεν εμφανιστούν τα παραπάνω configurations, θα πρέπει να δημιουργηθούν στο Intellij. 

(3) Για τα metadata των βίντεο χρησιμοποιήσαμε την βιβλιοθήκη tika και θα πρέπει να προστεθεί στο project.
	Κατεβάστε εδώ: https://downloads.apache.org/tika/tika-app-1.26.jar
	
(4) Κάποια βίντεο για να τρέξετε την εφαμογή μπορείτε να τα βρείτε εδώ: https://drive.google.com/file/d/1gKP6veMRkRh6DBDfYzf6VKaoAlJv3Wp6/view?usp=sharing
